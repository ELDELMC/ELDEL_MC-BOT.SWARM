🧩 Lo que se implementó / corrigió (storytime técnico):
Se detectó que el comando `.invo` no está respondiendo. Se realizó una auditoría completa:

**Análisis realizado:**
1. ✅ Validación de sintaxis: plugins/invo.js no tiene errores de parsing
2. ✅ Verificación de carga: El módulo se carga correctamente con export default
3. ✅ Estructura del handler: Firma correcta `async handler(sock, message, args, context)`
4. ✅ Flujo en MessageHandler: El comando se reconoce correctamente y debería ejecutarse

**Problema identificado:**
El comando tiene dos restricciones que podrían bloquear su ejecución:
- `adminOnly: true` → El usuario DEBE ser admin del grupo
- `groupOnly: true` → El comando solo funciona en GRUPOS

Sin embargo, hay un POTENCIAL PROBLEMA adicional: Si el usuario intenta usar `.invo` en un contexto donde el bot NO es admin del grupo, se ejecutará el delegador a otra sesión (línea 223-245 en MessageHandler.js). Si NINGUNA sesión es admin, el comando es rechazado silenciosamente.

**Solución implementada:**
Mejoré los mensajes de error y agregué logging más detallado para que el usuario sepa EXACTAMENTE por qué su comando no funciona.

📄 Código clave creado/modificado:
```javascript
// En plugins/invo.js - Handler modificado para mejor debugging:
async handler(sock, message, args, context) {
    const { chatId, senderId, isGroup, isBotAdmin, senderIsOwner, isSenderAdmin } = context;
    
    if (!isGroup) {
        await sock.sendMessage(chatId, {
            text: reply('❌ Este comando solo funciona en GRUPOS.'),
        }, { quoted: message });
        return;
    }
    
    if (!isBotAdmin && !senderIsOwner) {
        await sock.sendMessage(chatId, {
            text: reply('❌ El bot debe ser ADMINISTRADOR del grupo para invitar usuarios.'),
        }, { quoted: message });
        return;
    }
    
    // ... resto del handler
}
```

Estado actual del bot después de este cambio: Comando .invo ahora con mensajes de error más claros para debugging.

Se implementó un sistema de seguimiento de actividad por grupo. Cada vez que un usuario envía un mensaje (que no sea un comando y no venga del bot), el `ActivityTracker` incrementa su contador en `data/activity.json` utilizando operaciones atómicas a través de `sharedData.update`.

Se creó el comando `.top activos` que recupera los 10 usuarios con mayor cantidad de mensajes en el grupo actual, presentándolos en un ranking visualmente atractivo con medallas para el podio.

📄 Código clave creado/modificado:
```javascript
// core/ActivityTracker.js
async recordMessage(chatId, senderId, senderName) {
    await sharedData.update(DATABASE_FILE, {}, (data) => {
        if (!data[chatId]) data[chatId] = {};
        if (!data[chatId][senderId]) data[chatId][senderId] = { count: 0, name: senderName };
        data[chatId][senderId].count += 1;
        return data;
    });
}
```
Estado actual del bot después de este cambio: Ahora cuenta mensajes y tiene un ranking de actividad funcional.
